General info

Introduction

uROSnode is a middleware targeted to embedded systems which need to communicate with ROS, in a lightweight and fast way. It is developed in pure ANSI C, which makes it easily portable and optimizable by almost all the embedded system compilers.

Folders

The structure of the source package folders is the following:

  • /: Root folder, which contains Makefile sources to be included for your platform
  • demos: This folder contains some usage examples, developed for different target platform
    • demo-xxx ...
  • doc: Documentation stuff
    • doxygen: Resources for Doxygen generation
      • dot: Graphviz .dot files
      • dox: Doxygen .dox text-only files
      • img: Images to be included in the documentation
    • html: HTML user manual
  • include: Header files
  • misc: Miscellaneous stuff, for experimental purposes
  • src: Source code files
    • lld: Low-level drivers
      • driver-xxx ...
  • template: Template files
    • include: Template header files
    • src: Template source files
      • lld: Template low-level driver files

Modules

The software is written in a modular fashion. Their logical subdivision is the following, based on the abbreviated names:

  • urosBase: Basic data types and features (e.g. memory management)
  • urosConn: Connectivity features, platform-indipendent
  • urosNode: Node related stuff
  • urosRpc*: XMLRPC related stuff
    • urosRpcCall: XMLRPC call functions and data types
    • urosRpcParser: XMLRPC parser for incoming streams
    • urosRpcSlave: ROS XMLRPC Slave API management
    • urosRpcStreamer: XMLRPC writer for outgoing streams
  • urosTcpRps: TCPROS data types and management
  • urosThreading: Multithreading features, platform-independent

Additionally, some modules need low-level bindings to platform-dependent libraries, or callbacks for the project application:

  • uros_lld_base: Platform-dependent memory management features
  • uros_lld_conn: Bindings to the chosen low-level connectivity library
  • uros_lld_node: Node configuration load/save to non-volatile memory
  • uros_lld_threading: Bindings to the multithreading features of the chosen operating system
  • urosUser: User callbacks, application-dependent.
