uros_err_t uros_lld_conn_recv ( UrosConn *  cp,
void **  bufpp,
size_t *  buflenp 
)

Receives some data.

The connection object waits until some data is received from the remote address. The pointer to the received data is returned, with a length up to the requested one.

Warning
Not conceptually equivalent to POSIX connect(), as the buffer is not provided by the user, but by the middleware.
Precondition
The connection must be open and working.
Postcondition
Either data is received, or an UROS_ERR_EOF is returned when there is no more pending data (connection closed by remote).
Parameters
[in,out]cpPointer to a communicating connection object.
[out]bufppIndirect pointer to the received data buffer.
[in,out]buflenpPointer to the buffer length in bytes, with the following meaning:
  • At call, it indicates the maximum length of the received data.
  • At return, it is the actual length of the received data.
Returns
Error code.

uros_err_t uros_lld_conn_settcpnodelay ( UrosConn *  cp,
uros_bool_t  enable 
)

Sets the Nagle algorithm state.

When the Nagle algorithm is enabled, small packets are not immediately sent, but possibl enqueued, so that the TCP overhead is minimized. When it is disabled, even one-byte packets are immediately sent, to minimize latency despite the TCP overhead.

Note
Available only for TCP connections.
Parameters
[in,out]cpPointer to a valid UrosConn object.
[in]enableNagle algorithm activation switch.
Returns
Error code.
